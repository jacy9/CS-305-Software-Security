# CS-305-Software-Security
For this portfolio artifact, I selected the Artemis Financial Vulnerability Assessment Report because it demonstrates my understanding of software security principles and secure development practices. Artemis Financial is a financial consulting company that needed stronger security controls to protect sensitive customer financial information and support secure communications. My role was to identify vulnerabilities within the software application and recommend solutions that aligned with industry security standards. I performed a vulnerability assessment, analyzed dependency reports, and evaluated risks that could affect the confidentiality, integrity, and availability of data. I believe I did well in identifying security concerns and connecting them to practical mitigation strategies. This project reinforced the importance of secure coding because software vulnerabilities can expose sensitive information and increase organizational risk. Strong software security improves customer trust, protects company assets, and helps organizations meet security and compliance requirements.

The vulnerability assessment challenged me to think beyond functionality and focus on security throughout the software development lifecycle. The most valuable part of the project was learning how encryption, hashing, certificates, and dependency management work together to protect an application. I increased layers of security by implementing HTTPS communication with a self-signed certificate, generating SHA-256 checksums to verify data integrity, and recommending AES-256 encryption to protect sensitive information. I used OWASP Dependency-Check to identify vulnerable libraries and evaluate potential risks. After making security improvements, I tested the application to verify that the code remained functional and that the new security controls did not introduce additional vulnerabilities. In future projects, I would continue using vulnerability assessments, OWASP resources, dependency analysis tools, and secure coding standards to identify risks and select appropriate mitigation techniques. This artifact demonstrates my ability to assess software vulnerabilities, apply cryptographic solutions, implement secure communication protocols, and use security tools to improve application security. These skills represent practical experience that I can showcase to future employers as evidence of my knowledge of modern software security practices.
